Release Notes 4.31h

The prior versions were `4.1` and `4.2` and `4.30`.

As Of 06/30/2015

===Version 430+zmf===
Added final keepalive send before turning off the copy_sort_index_sys_cmd flag.<br>
Revised SSL shutdown wait loop.<br>
Set SSL shutdown start time (this was causing a reconnect timeout because it wasn't set).<br>

Changed close listener wait from 5 seconds down to 2 seconds.

===Version 431===
Made connection status global.<br>
Prevented sending KB char to server when reconnecting.<br>
Level 14 logging of client-server messages (debug version only).<br>

===Version 431a===
Error `413` message (window not on top) clarified.<br>
Suppressed logging in checkSSL until CS version match established.<br>
Polished up source terms and spelling errors - no object affect.<br>
Revised order of Menu Display responses to brserver.<br>
Allowed BR MSG() keyboard function to work when Command Console is the active window.<br>

===Version 431b===
Corrected exit pre-processing for both normal and unattended modes.<br>
Changed "blank" references to "blanksPtr".<br>
Completely revised UNATTENDED_INPUT_CALL processing.<br>
Changed HTTP "Client-Inquiry" to "PAGE-URL".<br>
Added HTTP support for "PAGE-HEADER".<br>

Revised error messages that pertain to system errors to indicate as much.<br>
Added some DLL errors to the BRERR$ definitions file. <br>

Added support for getting cookies via http. <br>
Cleaned up some errors causing dumps during termination.<br>
Switched log information to message boxes for errors that occur while uploading the client DLL.<br>

Revised SYSTEM LOGOFF to not fail to zero session IDs when issued to BRSERVER. 
This is to fix the problem where Windows reports active sessions where there is none, causing WSID's to be assigned incorrectly.

===br4.31ce===
Fixed the problem where NOWAIT was being ignored if OPTION 38 was on.

===Version 431d===
Removed DUPLICATE FLOWSTACK code.<br>
Revised ENDRUN processing of libraries .. set currentFileChain and currentScreenChain values during library PCB detachment. This fixed a problem detaching a NOFILES library.<br>
Lots of changes to run.cpp for flowstack processing and readability.

Changes from prior editions:<br>
*SQL and SYSERR fixes.
*Suppress error 867 on a radio button.
*Don't show LABELS, and BUTTONS in STATUS command when GUI is off.
*Removed an error trying to close an already closed window. 

===Version 431d===
Removed DUPLICATE FLOWSTACK code.<br>
Revised ENDRUN processing of libraries .. set currentFileChain and CurrentScreenChain values during library PCB detachment. This fixed a problem detaching a NOFILES library.<br>
Lots of changes to run.cpp for flowstack processing and readability.

Changes from prior editions:
*SQL and SYSERR fixes.
*Suppress error 867 on a radio button.
*Don't show LABELS, and BUTTONS in STATUS command when GUI is off.
*Removed an error trying to close an already closed window. 

Mismatched packets are ignored when:
*Ignore failure or keyboard return or we're terminating.
*Ignore failure is now on for cleanupNormalExits and closeWindowZero.
*When a flow stack error occurs, BR now returns error 9003.
*Current program name is stored on the client in a global when the status line is updated.

Corrected processing mismatched packets for CALL_FUNCTION_NULL_ACTION.<br>
Added transfer of current line number to client.

===Version 431e===
Revised remote.cpp ignoring of unmatched packets.<br>
Polished up the transfer of program name and line number to the client.<br>
Renamed clientOnly to ignoreFailureClientOnly<br>

Added error `0738` - SELECT should be a trailing attribute.<br>
This error occurs when SELECT is used as a leading attribute without the ^.

===Version 431f===
Added path to log message referencing brCrashDumpUploader.<br>
Added support for DEBUG INTERNAL COREDUMP_SERVER.<br>
Now we pass currentUserLineNumber to client in all models.<br>

===Version 431fb===
If an invalid trailing attribute is allowed via OPTION 38 BR was occasionally dumping.<br>
Keepalive timeouts are prevented when sitting in a message box.<br>

===Version 431fc===
Created and applied MINOR_IO_ERROR (valued at 3). This relates to log levels.

===Version 431fe===
Ignore mismatched packets on menu display.<br>
Suppress timeout during MSGBOX call.<br>
Fix 2D control header font colors.<br>
Removed V42 references.<br>
Fixed a problem re loading library modules RELEASE.<br> 
Corrected logging before DLL loaded.<br>

===Version 431ff===
Revised PIC processing.

===Version 431fg===
Revised PIC, and DATE processing to test formatted field capacity when inserting characters. (FMT runs only in overtype mode so it isn't affected.)<br>
Statusline ROWCOL now defaults to off.<br>

===Version 4.31g===
Upgrade to VS 2010.<br>
Fixed problem with license verification only to first decimal point of version number.<br>
Change made to prevent looping during shutdown.<br>
Client changed to let the server determine whether or not to terminate.<br>
Graceful client shutdown. See build notes for details.<br>

===Version 431ga===
Removed UPGRADE code flag from source.<br>
Fixed date problem that was created by recent FMT and PIC revisions.<br>
KREC is updated during printing only when the PUSH level is zero.<br>
For HTTP client, the log data showing CURL information was extended.<br>
The HTTP client now does a GET if no printing occurs prior to a LINPUT.<br>
A display parameter was added to the fldedit function. This function is used 
by FMT, PIC DATE and other field types to see if typing is done in excess of field capacity.<br>

===Version 4.31gc===
More detail is provided when a database connection attempt is unsuccessful.<br>
Corrected failures pertaining to deleting characters in non-currency PIC masks, especially with backspace. <br>
ENV$ and STATUS ENV search strings are now enhanced. See the wiki for details.
Internal documentation updated.<br>

===Version 4.31h===
Removed references to abort. Used CANCEL instead.<br>
Error 370 was dropping the program file during a REPLACE where OPTION 60 was on and the program contained 4.2+ statements. This is fixed.<br>
The XLATE system function was extended to support translation to and from UTF-8. <br>
Added error 438 "Input String not valid UTF-8 format."<br>
Removed system functions UPSI, LINEBUFFER, XPOS, CFIELDS$, and CHYRON$.<br>
Days(date("H:M:S"),"H:M:S") was producing a date with the first day of the current month. This was changed to the current day.<br>
RINPUT was positioning the cursor according to the previous Input. This was fixed.<br>
The Web Server support now does not tamper with binary data. It was converting to UTF-8.<br>
An internal pointer was adjusted to prevent crashes when the stack overflows.<br>
Default stack sizes were revised.
 work_size = 500000,    /* work stack size */
 flow_size = 100,       /* flow stack size */
 rpn_size = 1000,       /* rpn stack size */
 for_size = 100;        /* for-next stack */
