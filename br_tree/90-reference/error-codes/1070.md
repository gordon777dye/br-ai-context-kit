**Error:** 1070

**Description:** Invalid `Variable` name

**Details:** - Undefined variable name. A variable name was specified that was either invalid or not defined. One situation where this error can occur is when a FORM variable is not defined. In the following example, the variable P is used to identify the POS position. If P is not previously defined, this error will occur during execution: PRINT USING "FORM POS P, N 12.2": D
- `Break` used in conjunction with an otherwise unused variable or `internal function` (like `fkey`)
- Break Off attempted for a variable that did not have break turned on #Define the variable.
- Check the spelling of the variable.
- if automating break points use a temp variable to remember if you turned break on for something so you can test if it needs break to be turned off before attempting to turn it off.
